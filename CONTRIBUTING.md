# Contributing to BinBuz

Thanks for your interest. BinBuz is small and meant for internal/private use. If you want to contribute:

1. Fork the repo.
2. Create a branch: `git checkout -b feature/my-feature`
3. Add clear commits and a concise PR description.
4. Keep changes minimal and well documented.
5. If adding new dependencies, update `requirements.txt`.

Be mindful of client data and privacy when testing. Do not upload real client BIN data to public forks or issues.
